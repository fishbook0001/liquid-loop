# 零 LLM 依赖的 Agent 自组织记忆：液环原理与实现

## 🔥 所有 Agent 都在犯同一个错误

当前主流 Agent 记忆方案有一个共同特征：

**需要大模型来管理记忆。**

- All-Mem / MemGPT / Letta → 用 LLM 做 Split/Merge/Update
- CoreMem / AtomMem → 向量检索 + 外部评分排序
- MemPro / HeLa-Mem → LLM 摘要 + 压缩
- 结构化 Schema → LLM 设计 + 维护

问题是：**大模型来管记忆，等于用 CEO 来扫地。**

每次记忆操作都要调用 LLM —— 500~2000ms 延迟，GPT-4 级别成本。那如果 LLM 不可用呢？Agent 的记忆就僵住了。

---

## 💡 第三条路：自组织记忆

液环（Liquid Loop）从经典物理结晶过程获得灵感：

> 晶种（Anchor）在过饱和溶液中自动吸引同质粒子（Evidence），当粒子积聚到阈值，自动结晶（Memory）。

整个过程**不需要外部做功**——证据在锚点周围自然聚集，形成有密度的"认知团"。

---

## ⚡ 三行代码感受一下

```python
from liquid_loop import WorkspaceState

state = WorkspaceState()
aid = state.add_anchor("用户偏好", "简洁优先")
state.add_evidence(aid, "用户偏好简洁输出")   # 第 1 次
state.add_evidence(aid, "用户偏好简洁输出")   # 第 2 次 → 自动结晶
```

执行完你就有了：
- 1 个锚点（用户偏好）
- 2 条证据（含 1 条重复触发结晶）
- 1 个记忆（置信度 1.00）
- 四维熵值 0.0175（绿色健康）

CLI 也行：

```bash
pip install liquid-loop
cd my-project
liquid-loop init
liquid-loop anchor-add "用户偏好" "简洁优先"
liquid-loop evidence-add "用户偏好" "用户偏好简洁输出"
liquid-loop evidence-add "用户偏好" "用户偏好简洁输出"
liquid-loop status
# → 锚点: 1 | 证据: 2 | 记忆: 1 | 熵值: 0.0175 [GREEN]
```

## 🧪 比 LLM 记忆快 500 倍

| 数据量 | 写入 | 成核 | 熵值 | 产出记忆 |
|--------|------|------|------|----------|
| 1,000 | 16 ms | 2 ms | 0.1 ms | 23 |
| 10,000 | 134 ms | 19 ms | 0.6 ms | 205 |
| **50,000** | **683 ms** | **129 ms** | **3.5 ms** | **479** |

吞吐 ~73K 写入/秒，单条延迟 ~0.014ms。**比 LLM 调用快 500 倍以上。**

---

## 🛡️ 天然抗噪

在 0% / 20% / 50% 三种噪声水平下，熵值完全相同（0.1935）。

为什么？液环使用**精确字符串匹配**成核，不依赖语义相似度。噪声只是增加证据量，不影响结晶质量。（语义匹配已规划为可选扩展。）
